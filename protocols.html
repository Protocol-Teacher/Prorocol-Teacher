Network Protocols

Ethernet II

Ethernet II frame is a basic data unit that is transported in the physical layer of the OSI model.
It includes the following header fields:

- Preamble: Preamble helps with synchronization between both the sender and receiver.

- Destination Address: The MAC address of the receiving node.

- Source Address: The MAC address of the sending node.

- Type: Includes the type of packet in its data field.
Type Codes:
IPv4 080016
IPv6 86DD16

- Payload (Body): A maximum payload of 1500 octets.

- CRC: Cyclic Redundancy Check helps with error detecting.



IPv4

IPv4 is the most used internet protocol, it is a connectionless protocol, it does not guarantee delivery, nor data integrity. It uses an IPv4 address space of 31 bits, which is not enough to give each node a unique address, this problem is solved by using the later version IPv6 which uses an address space of 128 bits.
It includes the following header fields:

- Version: This includes a four-bit version field, it is always set to 4

- HLen: Internet Header Length includes the size of the header in four bits.

- TOS: The Type of Service is used to request a low delay route and priority.

- Length: This includes the size of the packet including the header. Maximum size is 65,535 bytes and minimum size is 20 bytes.

- Ident: This identifies the fragments of an IP datagram.

- Flags: This is used to identify fragments. If the second bit is set then the packet will not be fragmented, and it will be dropped if it can not be routed without fragmentation. If the third bit is set then more fragments are expected, if not set then it will be the last fragment.
bit 0: Reserved; must be zero.
bit 1: Don't Fragment
bit 2: More Fragments

- Offset: This is used to place the fragmented packet in the correct position within the entire unfragmented packet. The value is multiplied by 8 which gives the maximum size of 212 Ã— 8 = 65,528 bytes.

- TTL: Time to Live helps prevent packets from circling the internet indefinitely. It also helps with counting hops since every time a packet reaches a router the TTL is decreased until it reaches zero then the packet gets dropped.

- Protocol: This includes the protocol carried in the data section of the IPv4 packet.

- Checksum: Checksum is used for error checking.

- SourceAddr: Includes the IPv4 address of the source node.

- DestinationAddr: Includes the IPv4 address of the destination node.

- Options (variable): This field is not usually used.

- Pad (variable): This is used to ensure that the packet length is a multiple of 32 bits.

- Payload: This includes the data carried in the IPv4 packet.



IPv6

IPv6 was introduced to solve problems with IPv4. The main problem is the number of addresses available, IPv6 has an address space of 128 bits, this means it can support an astronomical number of devices (340 undecillion). It also provides other technical benefits.
It includes the following header fields:

- Version: Include a constant value of 6.

- TrafficClass: This includes two main functionalities. The first 6 bits are called Differentiated Services, it is used to classify packets. The last 2 bits are called Explicit Congestion Notification, it is used for congestion control.

- FlowLabel: This is used to make sure that the packet stays on the same path when set to non-zero.

- PayloadLen: This includes the size of the payload carried in octets.

- NextHeader: This includes the protocol carried in the payload section of the IPv6 packet.

- HopeLimit: Hope Limit prevent packets from circling the internet indefinitely. It also helps with counting hops since every time a packet reaches a router the HopeLimit is decreased until it reaches zero then the packet gets dropped.

- SourceAddress: Includes the IPv6 address of the source node.

- DestinationAddress: Includes the IPv6 address of the source node.

- Next Header/Data: This includes the data carried in the IPv6 packet.



TCP

The Transmission Control Protocol is one of the most commonly used protocols. It provides error-checking, reliability, and ordering for applications running on different devices. Most notably the World Wide Web, file transfer, email, and more.
It includes the following header fields:

- SrcPort: Includes the sending port.

- DstPort: Includes the receiving port.

- SequenceNum: This includes two main functionalities:
When the SYN flag is set then it will include the initial sequence number.
When the SYN flag is not set then it will include the accumulated sequence number.

- Acknowledgment: When the ACK flag is set this will include the next sequence number.

- HdrLen: This includes the length of the header.

- 0

- Flags: This includes 9 1-bit flags.

- AdvertisedWindow: This includes the receive window size.

- Checksum: Checksum is used for error checking.

- UrgPtr: When URG flag is set then this will include an offset from the sequence number.

- Options (variable): This includes; Option-Kind, Option-Length, and Option-Data.

- Data: This includes the data carried in the packet.



UDP

The User Datagram Protocol is another commonly used protocol. It is a simple and connectionless protocol. It does not have guarantee of delivery, duplicate protection, or ordering. It is used when we only care about time delivery like with VoIP (Voice Over IP).
It includes the following header fields:

- Source Port: Includes the sending port.

- Destination Port: Includes the receiving port.

- Length: This includes the length of the header.

- Checksum: Checksum is used for error checking.

- Data: This includes the data carried in the packet.



ICMP

The Internet Control Message Protocol (ICMP) is an error checking protocol. It helps devices report errors to each other through a network. For example, it can be used to report that a host or a router cannot be reached.
It includes the following header fields:

- Type: Includes the ICMP type.

- Code: Includes the ICMP subtype.

- Checksum: Checksum is used for error checking.

- Identifier: Used with a timestamp request.

- Sequence Number: Used with a timestamp request.

- Optional Data: Includes a copy of the IP header, and the first 8 bytes of the data.
